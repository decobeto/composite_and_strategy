Design Patterns

Composite

Um padrão estrutural que permite compor objetos em forma de árvore e então trabalhar com essas estruturas como se fossem objetos individuais. Entretanto esse padrão só faz sentido se o modelo da aplicação for representado em forma de árvore.

Um exemplo disso é a função de camadas do Adobe Photoshop.

Funcionamento

A ideia principal é trabalhar com hierarquias através de uma interface comum, ou seja, uma classe complexa ou apenas um objeto simples são tratados pela mesma interface.

Estrutura

1. Interface

A interface do componente descreve operações tanto para elementos simples como para elementos complexos da árvore.

2. Folha

Componente de uma árvore que não tem sub-elementos, responsáveis pela maior parte do trabalho.

3. Contêiner ou Composite

Elemento que possui sub-elementos, sendo esses folhas ou outros contêineres. Assim atuando como um interface, ao receber um pedido o mesmo envia o trabalho para um de seus sub-elementos e apenas retorna o resultado final.

4. Cliente

Responsável por trabalhar com todos os elementos através da interface.

Vantagens

  1. É possível trabalhar com estruturas de árvore mais complexas de forma conveniente usando polimorfismo e recursão a seu favor.
  2. É possível introduzir novas funcionalidades ao código sem quebrar as já existentes .

Desvantagens

  1. A interface principal do componente precisa ser muito genérica para interagir com diferentes funcionalidades.
